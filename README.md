# DreamAI — AI-generated 3D worlds

Type a prompt → Qwen designs the scene → FLUX paints reference images → TRELLIS turns them into 3D models → Three.js renders an explorable world in your browser.

## Live demo

GitHub Pages: https://necktomor.github.io/DREAMAIBETA/

Controls:
- **WASD / arrows** — move
- **Mouse** — look around
- **Shift** — sprint
- **Space** — jump
- **ESC** — release cursor

## Architecture

```
prompt
  │
  ▼
Qwen LLM ──► scene.json (biome, buildings, nature, layout)
  │
  ▼
Gemma planner ──► world positions, rotations, spawn
  │
  ▼
FLUX.1 ──► reference PNG per hero object
  │
  ▼
TRELLIS (Replicate A100 → fal.ai fallback) ──► GLB
  │
  ▼
trimesh normalize + simplify ──► web/glb/*.glb
  │
  ▼
Three.js viewer (web/main.js) — cel-shaded, bloom, instanced trees
```

## Local development

### Static viewer only (just look at a generated scene)

```bash
python3 -m http.server 8765 --directory web
# open http://localhost:8765
```

### Live pipeline (generate new scene from prompt)

```bash
pip install -r requirements.txt
cp .env.example .env
# fill in your tokens

python main.py "Foggy ocean cove with a wooden lighthouse"
# scene.json + glb/ written into web/
python web_server.py
# open http://localhost:8080
```

## Required env vars

| Variable | Where |
|---|---|
| `QWEN_API_KEY` | alem.ai (Qwen) |
| `GEMMA_API_KEY` | alem.ai (Gemma planner) |
| `REPLICATE_API_TOKEN` | Replicate (FLUX + TRELLIS A100) |
| `FAL_KEY` | fal.ai (TRELLIS-2 fallback) |
| `HF_TOKEN` | HuggingFace (optional, mirror tokens) |

**Never commit `.env`.** Use `.env.example` as a template.

## Repo layout

```
DREAMAIBETA/
├─ main.py              # CLI entry: python main.py "prompt"
├─ web_server.py        # dev server (live pipeline mode)
├─ deploy_web.py        # legacy compat
├─ pipeline/
│  ├─ orchestrator.py   # end-to-end runner
│  ├─ qwen_client.py    # prompt → scene JSON
│  ├─ planner_client.py # AI layout
│  ├─ flux_client.py    # PNG generation
│  ├─ replicate_client.py
│  ├─ fal_trellis_client.py
│  ├─ glb_normalizer.py # scale + simplify to ≤50k faces
│  └─ layout_fixer.py   # rule-based layout fallback
└─ web/
   ├─ index.html
   ├─ main.js           # Three.js renderer (toon, bloom, water shader)
   ├─ scene.json        # current world definition
   └─ glb/              # 3D models
```

## Tech

- Three.js 0.170 — WebGL renderer
- MeshToonMaterial with 4-band gradient — cel-shading
- UnrealBloomPass — gentle glow on bright sources (moon, fireflies, lanterns)
- Custom ShaderMaterial — vertical sky gradient + animated water waves
- InstancedMesh / clone pools — efficient forest of 250+ trees
- Procedural terrain — multi-octave sine noise + flatten-zones under buildings
